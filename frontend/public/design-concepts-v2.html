Handling LLM Disagreement in Matrix View

When Gemini and ChatGPT disagree on who wins a competitive question, how should we display this in the heatmap?

1 Simple Aggregation with "Split" State

Three-state system: Won (both agree), Lost (both agree), Split (disagree)

Recommended Simplest
How it works

Won (Green): Both LLMs chose your brand as #1
Lost (Red): Both LLMs chose a competitor as #1
Split (Amber): LLMs disagree - one chose you, one chose competitor

Cell shows the winner when both agree, or "Split" with both names when they disagree.

2 Separate Sub-Rows per LLM

Each question has two rows - one for Gemini, one for ChatGPT

Most Detailed
How it works

Each question expands into two sub-rows showing each LLM's individual verdict.
This preserves full detail but doubles the vertical space needed.
Best when you need to compare LLM-specific patterns (e.g., "ChatGPT prefers Nike more than Gemini does").

3 Dual-Bar Cells (Side-by-Side)

Each cell shows both LLM verdicts as mini bars

Compact Detail
How it works

Each cell contains two mini-bars side by side: left = Gemini, right = ChatGPT.
Color indicates won (green) or lost (red) for each LLM independently.
Compact but information-dense - shows both verdicts in single cell space.

4 Win Rate Percentage Bar

Show confidence level: 100% = both agree, 50% = split

Quantitative
How it works

Each cell shows a win percentage based on LLM agreement:
100% won = Both LLMs chose you
50% won = One LLM chose you, one chose competitor
0% won = Both LLMs chose competitor

Progress bar visualizes the confidence level at a glance.

My Recommendation

Strategy 1 (Simple Aggregation with Split State) provides the best balance of clarity and information density. Users can quickly scan for patterns, and the amber "Split" state clearly signals uncertainty without overwhelming detail. For power users who want LLM-specific breakdown, add a hover tooltip or click-to-expand that shows Strategy 3's dual-bar detail.

Strategy 1 + Hover Detail = Best of Both Worlds
